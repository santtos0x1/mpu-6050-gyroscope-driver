# Contributing

Thank you for considering contributing to this project.

The goal of this repository is to provide a clean, lightweight driver and utilities for working with the MPU6050 sensor on the RP2040. Contributions that improve reliability, readability, and usability are welcome.

## How to Contribute

1. Fork the repository.
2. Create a new branch for your change.

```
git checkout -b feature/my-feature
```

3. Make your changes.
4. Commit your changes with a clear message.

```
git commit -m "feat: add sensor calibration function"
```

5. Push your branch.

```
git push origin feature/my-feature
```

6. Open a Pull Request.

Please describe clearly what your change does and why it is needed.

---

## Types of Contributions

Contributions may include:

* Bug fixes
* Performance improvements
* Code refactoring
* Documentation improvements
* New examples
* Additional hardware support

---

## Code Style

To keep the project consistent:

* Follow the existing naming conventions
* Keep functions small and focused
* Write clear comments where needed
* Avoid unnecessary dependencies
* Avoid SDK dependencies
* Keep the code portable and lightweight

---

## Reporting Bugs

If you find a bug, please open an Issue and include:

* Description of the problem
* Steps to reproduce
* Expected behavior
* Actual behavior
* Hardware used
* Relevant logs or outputs

---

## Feature Requests

Feature requests are welcome. Please open an Issue describing:

* What you want to add
* Why it would be useful
* Possible implementation ideas (if you have them)

---

## Pull Request Guidelines

Before submitting a Pull Request:

* Ensure the code builds correctly
* Test the changes if possible
* Keep the PR focused on one change
* Update documentation when necessary

---

## Respect the Community

Please follow the project's Code of Conduct and maintain respectful communication when discussing issues or pull requests.
